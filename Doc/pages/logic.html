Logical Expressions

So we now know enough to be able to show parts of a tiddler based on the contents of a variable that holds a truth value. But what if we want to show a choice if the main character doesn't have an empty gun?

:: A showdown
<<if $bullets gt 0>>

<<Fire the gun>>

<<endif>>

There are a whole bunch of logical operators you can use to compare values, ending up with a truth value:

eq Evaluates to true if both sides are equal. $bullets eq 5
neq Evaluates to true if both sides are not equal. $friends neq $enemies
gt Evaluates to true if the left side is greater than the right side. $money gt 3.75
gte Evaluates to true if the left side is greater than or equal to the right side. $apples gte $carrots + 5
lt Evaluates to true if the left side is less than the right side. $shoes lt $people * 2
lte Evaluates to true if the left side is less than or equal to the right side. 65 lte $age
and Evaluates to true if both sides evaluates to true. $hasFriends and $hasFamily
or Evaluates to true if either side is true. $boy or $girl
not Flips a true value to a false value, and vice versa. (not $hungry) or ($location eq "restaurant")

With these you can write out pretty complicated conditions. Parentheses are your friend here. They work exactly the same way they do in other kinds of expressions -- they force things to be evaluated in the order you want them to.

<<if ($walkingHome or ($sandwiches gt 7)) and (not $breathing)>>

Remember that you can test these out with the Control-Alt-P shortcut at any point in your story.

This section is probably confusing if you've used JavaScript before. The reason for all these new operators is that the > character confuses TiddlyWiki's parser, so an equivalent needed to be created. The rest are simply there for the sake of parallelism, and also to cut down on the inevitable = vs. == confusion. You can still use !=, for example, if you want to.
