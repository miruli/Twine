Tweebox in 30 Seconds

This page goes over the basics of using Tweebox to compile a TiddlyWiki. It doesn't try to explain anything except what clicking things does. Building Simple Stories, on the left, is a much better general introduction.

The Projects toolbar appears directly underneath tweeBox's title. A project is a collection of source files and settings. Switch projects with the pull-down menu on the left side of the toolbar. Add and remove projects with the buttons on the right side.

Below the Projects toolbar is information about the current project.

The Source Files section shows the Twee source files currently part of the project. Add a file by clicking the Browse button at the bottom of the list. Remove it by clicking the minus-sign button beside the file.

The Save Story As section lets you choose where the compiled TiddlyWiki file will be saved. You may only save to the same folders that your source code is in. If you have source files spread across several folders, you can choose from them with a drop-down menu. Type in the file name you'd like to use. This name must end in .htm or .html.

The Author name field contains the author's name — not all kinds of TiddlyWikis show this information, however. The Story Format menu lets you choose what kind of TiddlyWiki to create. Choices include:

  • Jonah: a standalone format intended for Choose Your Own Adventure-style stories.
  • TiddlyWiki 2 and TiddlyWiki: standard formats created by Jeremy Ruston.

Click the Build button to compile your Twee files. Once it finishes, Tweebox tries to open your created file in a new window. Often, pop-up blockers will prevent this from happening; a small notification box will appear in the bottom-right corner of the screen with a button to view your new TiddlyWiki.
